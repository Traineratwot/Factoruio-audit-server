# Factorio Audit
Laravel 13 + React 19 (Inertia) app that audits Factorio mods via the Factorio Mod Portal API.

## Stack

- **Backend**: Laravel 13, PHP 8.3+, PostgreSQL 18, Redis, Meilisearch
- **Frontend**: React 19, Inertia.js v3, TypeScript, Tailwind CSS v4, Vite 8
- **Queue**: Laravel queues with Supervisor in Docker
- **Search**: Meilisearch via Laravel Scout
- **Admin**: Filament v5
- **Testing**: Pest v4, PHPUnit
- **Linting**: ESLint + Prettier (JS), Laravel Pint + PHPStan level 7 (PHP)
- **Docker**: Multi-stage Dockerfile (dev/prod), docker-compose.dev.yml

## Factorio Mod Portal API

https://wiki.factorio.com/Mod_portal_API

## Database schema

### mods
- `id`, `name` (unique), `owner`, `latest_version`, `category`, `title`, `summary`
- `downloads_count` (int), `popularity` (float)
- `thumbnail`, `description`, `homepage`, `license` (json), `tags` (json), `images` (json)
- `changelog`, `score` (float), `factorio_version`, `latest_release_date` (timestamp)
- `author_id` (FK → authors)
- `releases` (json, legacy — data moved to mod_versions)

### reports
- `id`, `mod_id` (FK → mods), `mod_version`, `score` (float), `raw` (jsonb), `sha1` (char 40, unique)
- Unique constraint on `(mod_id, mod_version)`

### mod_versions
- `id`, `mod_id` (FK → mods), `version`, `file_name`, `download_url`, `sha1`
- `factorio_version`, `dependencies` (json), `released_at` (datetime)

### authors
- `id`, `name`

### users
- Standard Laravel users table (used by Filament admin)

## Key architecture notes

- **AuditService** connects to an external WebSocket server (`AUDIT_WEBSOCKET_URL`) using JSON-RPC 2.0 to scan mods. This is a separate service, not part of this repo.
- **FactorioService** fetches mod list from `https://mods.factorio.com/api/` and caches results for 1 hour.
- **Mod::getImage()** returns `https://assets-mod.factorio.com{thumbnail}` or null if no thumbnail.
- **Mod::fetchFullInfo()** calls FactorioService to populate thumbnail, releases, versions, etc. Used as fallback when a mod has no versions.
- **Mod::runAudit(?string $version)** dispatches to AuditService via WebSocket, creates/updates Report.
- Generated code is in `resources/js/actions/`, `resources/js/routes/`, `resources/js/wayfinder/` (gitignored, auto-generated by Laravel Wayfinder).

## Routes

```
GET  /                                    → ModController@index (catalog)
GET  /search/mods                         → ModController@search (Meilisearch)
GET  /report/mod/{mod}                    → ModController@report
GET  /report/mod/{mod}/version/{version}  → ModController@report
GET  /api/mods/search                     → AuditController@search (DB LIKE fallback)
GET  /api/mods/{mod}/versions             → AuditController@versions (fetches full info if empty)
POST /audit                               → AuditController@store (rate limited, dispatches AuditJob)
```

## Audit system

- **Rate limiting**: 5 audits/hour per IP for unauthenticated users. Authenticated users (logged in via `/admin`) are exempt.
- **AuditJob** accepts `$modId` and optional `$version`, dispatches to `work1` queue.
- **AuditController@store** validates `mod_id` + `mod_version`, checks rate limit, dispatches AuditJob.
- **AuditController@versions** returns mod versions; if empty, calls `Mod::fetchFullInfo()` to populate them.
- **AuditController@search** provides DB-based mod search (fallback when Meilisearch is unavailable).

## Frontend pages

### Welcome (catalog)
- Mods table with search, sorting, pagination
- Sidebar: ReportFilter (all/with/without report), CategoryFilter (include/exclude)
- "Audit Mod" button in header opens AuditDialog
- Per-row "Audit" button (when reports_count=0) opens AuditDialog with mod pre-selected

### Report
- Version switcher dropdown with visual indicators (green check = has report, gray clock = no report)
- Selecting a version with report → navigates to it
- Selecting a version without report → opens AuditDialog with mod + version pre-selected
- Mod image shown in header (from Mod::getImage())
- Stats, progress bar, scanner results, findings tables

## Frontend conventions

- **PrimeReact** components: DataTable, Dialog, Dropdown, Card, Button, ProgressBar, etc.
- **Inline styles** used alongside Tailwind classes (project convention)
- **Dark theme**: `lara-dark-cyan` theme CSS imported in pages
- **Inertia.js** for navigation and data passing (no API calls for page loads)
- **CSRF**: Frontend reads `XSRF-TOKEN` cookie for POST requests

## ESLint conventions

- **Curly braces required** on all control statements (1tbs style)
- **Padding lines** around `if`, `return`, `for`, `while`, `do`, `switch`, `try`, `throw`
- **Type imports**: `import type { X }` enforced
- **Import order**: builtin → external → internal → parent → sibling → index, alphabetized
- Ignore: `vendor/`, `node_modules/`, `public/`, `resources/js/actions/`, `resources/js/components/ui/`, `resources/js/routes/`, `resources/js/wayfinder/`

## Environment

- `.env.local` takes priority over `.env` in Docker Compose
- Dev services: PostgreSQL, Redis, Meilisearch, MailHog, MinIO (object storage)

## Services

- `App\Services\FactorioService` – client for Factorio mods portal API
- `App\Services\AuditService` – client for mod audit tool (WebSocket JSON-RPC)

## Known limitations

- **Mod thumbnails**: not all mods have thumbnails populated (those that had `fetchFullInfo()` called). Most mods show letter avatar fallback.
- **Report uniqueness**: `(mod_id, mod_version)` is unique — one report per mod version.
