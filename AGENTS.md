# Factorio Audit
Laravel 13 + React 19 (Inertia) app that audits Factorio mods via the Factorio Mod Portal API.

## Stack

- **Backend**: Laravel 13, PHP 8.3+, PostgreSQL 18, Redis, Meilisearch
- **Frontend**: React 19, Inertia.js v3, TypeScript, Tailwind CSS v4, Vite 8
- **Queue**: Laravel queues with Supervisor in Docker
- **Search**: Meilisearch via Laravel Scout
- **Admin**: Filament v5
- **Testing**: Pest v4, PHPUnit
- **Linting**: ESLint + Prettier (JS), Laravel Pint + PHPStan level 7 (PHP)
- **Docker**: Multi-stage Dockerfile (dev/prod), docker-compose.dev.yml

## Factorio Mod Portal API

https://wiki.factorio.com/Mod_portal_API

## Key architecture notes

- **AuditService** connects to an external WebSocket server (`AUDIT_WEBSOCKET_URL`) using JSON-RPC 2.0 to scan mods. This is a separate service, not part of this repo.
- **FactorioService** fetches mod list from `https://mods.factorio.com/api/` and caches results for 1 hour.
- Generated code is in `resources/js/actions/`, `resources/js/routes/`, `resources/js/wayfinder/` (gitignored, auto-generated by Laravel Wayfinder).

## Commands

### PHP (run inside Docker container)

```bash
# Full lint + typecheck + test (CI check)
composer ci:check

# Lint only (Pint)
composer lint           # fix
composer lint:check     # check only

# PHPStan static analysis
composer types:check

# Tests (Pest)
composer test           # runs lint:check + types:check + artisan test

# Single test file
docker compose -f docker-compose.dev.yml exec -e APP_ENV=testing app php artisan test tests/Feature/SomeTest.php

# Clear config cache before running tests
php artisan config:clear --ansi
```

### JS/TS

```bash
# Lint
npm run lint            # fix
npm run lint:check      # check only

# Format
npm run format          # fix
npm run format:check    # check only

# Type check
npm run types:check

# Dev server
npm run dev
```

### Docker

```bash
# First-time setup
make init               # bun install + copy env + build + migrate + seed

# Start/stop
make up                 # start containers
make down               # stop containers
make restart            # restart containers

# Rebuild from scratch
make remake             # down + up + connect-db + migrate + cache + seed + cron + link

# Common tasks inside container
make bash               # shell into app container
make psql               # PostgreSQL console
make tinker             # Laravel Tinker
```

```
npm run lint:check → npm run format:check → npm run types:check → php artisan test
```

## ESLint conventions

- **Curly braces required** on all control statements (1tbs style)
- **Padding lines** around `if`, `return`, `for`, `while`, `do`, `switch`, `try`, `throw`
- **Type imports**: `import type { X }` enforced
- **Import order**: builtin → external → internal → parent → sibling → index, alphabetized
- Ignore: `vendor/`, `node_modules/`, `public/`, `resources/js/actions/`, `resources/js/components/ui/`, `resources/js/routes/`, `resources/js/wayfinder/`

## Prettier config

- Single quotes, semicolons, tab width 4 (YAML: 2)
- Tailwind CSS class sorting enabled via `prettier-plugin-tailwindcss`

## Testing

- Pest v4 with Laravel plugin
- Tests use SQLite `:memory:` database
- `APP_ENV=testing` is set in phpunit.xml
- Run tests in Docker: `make test` (sets `APP_ENV=testing`)

## Git hooks

- Pre-commit hook runs `vendor/bin/pint --dirty` and stages changed files
- Install: `make install-hooks`

## Environment

- `.env.local` takes priority over `.env` in Docker Compose
- Dev services: PostgreSQL, Redis, Meilisearch, MailHog, MinIO (object storage)
- `AUDIT_WEBSOCKET_URL` must point to an external audit scanner WebSocket server
