# Plan: Scanner version cron + outdated scanner warning banner

## Overview

Two features:
1. **Hourly cron** fetches current scanner version via `AuditService::scannerVersion()` and caches it
2. **Frontend banner** on report page warns when report was made with outdated scanner version, with re-scan button

## 1. Backend: `scanner:version` artisan command

**New file**: `app/Console/Commands/Scanner/VersionCommand.php`

```
signature: scanner:version
```

- Calls `AuditService::scannerVersion()`
- Stores result in cache: `Cache::put('scanner:current_version', $version, 86400)`
- Outputs the version to console

## 2. Schedule: hourly in `routes/console.php`

Add one line:
```php
Schedule::command('scanner:version')->hourly();
```

## 3. `ModController::report()` — pass scanner version to frontend

In `app/Http/Controllers/ModController.php:report()`:

```php
$currentScannerVersion = Cache::get('scanner:current_version');
```

Add to Inertia render props:
```php
'current_scanner_version' => $currentScannerVersion,
```

If cache is empty (first run before cron fires), the frontend will receive `null` and skip the comparison — no false warnings.

## 4. Frontend: TypeScript types

**File**: `resources/js/types/mod.ts`

Add `current_scanner_version` to `AuditReportViewerProps` (or wherever the report page props are typed).

## 5. Frontend: scanner-outdated warning banner

**File**: `resources/js/pages/report.tsx`

### New computed value (near existing `isOutdated`):

```ts
const isScannerOutdated =
    report !== null &&
    current_scanner_version !== null &&
    auditReport.scannerVersion !== current_scanner_version;
```

### New banner (after existing outdated-version banner, before stat cards):

Amber/yellow styled banner matching existing pattern:
- Icon: `pi pi-exclamation-triangle`
- Text: "This report was made with scanner vX, current version is vY"
- Button: "Re-audit v{current_version}" — opens AuditDialog with current mod version pre-selected
- Uses same AuditDialog instance already on the page (set `auditVersion` + `auditDialogVisible`)

### Re-scan button behavior:

Clicking "Re-audit" opens AuditDialog with:
- `preselectedMod`: `{ id: mod.id, name: mod.name, title: mod.title }`
- `preselectedVersion`: `current_version` (the same version the report is for)

This dispatches `POST /audit` with `mod_id` + `mod_version`, same as existing audit flow.
