# Mobile Adaptation Plan

## Problem
The app has hardcoded widths, fixed flex layouts, and a 7-column DataTable that overflow on mobile screens (<768px).

## Changes

### 1. `welcome.tsx` — Responsive sidebar layout
**Lines 130-178**: Change flex layout to stack on mobile.

```
Before: display: flex, sidebar width: 250px (always)
After:  display: flex, flex-col on mobile, flex-row on md+, sidebar hidden on mobile (use drawer or inline filters)
```

Approach: On mobile (<768px), hide sidebar filters behind a toggle button. Show filters inline below the header when toggled. On desktop, keep current sidebar layout.

- Add `useState` for filter visibility on mobile
- Add a "Filters" toggle button visible only on mobile (`md:hidden`)
- Wrap sidebar in a container that's `hidden md:block` by default, shown when toggled on mobile
- Reduce Container padding on mobile: `padding="1rem md:1.5rem"`

### 2. `welcome.tsx` — Responsive title
**Lines 111-124**: Reduce h1 font size on mobile.
- Change from fixed `fontSize: "2.5rem"` to responsive: `fontSize: "clamp(1.5rem, 5vw, 2.5rem)"`

### 3. `ModsTable.tsx` — Responsive DataTable
**Lines 87-154**: The table with 7 columns and `minWidth: "50rem"` overflows on mobile.

Approach: Hide less important columns on mobile, reduce minWidth.
- Change `minWidth` from `"50rem"` to `"40rem"` 
- Hide `Category`, `Downloads`, `Popularity`, `Added` columns on mobile using Tailwind classes on `<Column>` (PrimeReact columns don't support responsive classes directly, so use `className` with `hidden md:table-cell`)
- Keep: Name, Score, Actions (3 columns minimum)
- Remove `resizableColumns` on mobile (touch-unfriendly)

### 4. `TableHeader.tsx` — Responsive search
**Lines 23-86**: Search input fixed at `width: "20rem"`.

- Change search input width from `width: "20rem"` to `width: "100%"` with `maxWidth: "20rem"` so it fills available space on mobile
- Stack search + buttons vertically on mobile: use `flex-col sm:flex-row`
- Hide "Audit Mod" button label on mobile, show icon only

### 5. `report.tsx` — Responsive warning banners
**Lines 392-460**: Warning banners use `flex justify-between` which puts text and button side-by-side.

- Change to `flex-col sm:flex-row` so button wraps below text on mobile
- Reduce Container maxWidth to be more responsive

### 6. `report.tsx` — Responsive ModHeader
**Lines 171-220**: ModHeader uses `flex items-center gap-4`.

- Add `flex-col sm:flex-row` to stack avatar + info vertically on very small screens
- Reduce avatar size on mobile (pass responsive size or use CSS)

### 7. `AuditDialog.tsx` — Responsive dialog width
**Line 114**: Fixed `width: "500px"`.

- Change to `width: "min(500px, 90vw)"` so it fits on mobile screens

### 8. `PopularityColumn.tsx` — Hide stars on mobile
**Lines 15-37**: 5 stars take too much space.

- Hide the stars on mobile, show only the number
- Use `className="hidden sm:inline-flex"` on the stars container

### 9. `app.css` — Responsive paginator
**Lines 31-66**: Paginator buttons have `min-width: 2.5rem`.

- Reduce paginator button sizes on mobile
- Add media query: `@media (max-width: 640px) { .p-paginator .p-paginator-first, ... { min-width: 2rem; height: 2rem; } }`

### 10. `Column` responsive visibility
Create a helper or use PrimeReact's `className` prop to hide columns:
- `className="hidden md:table-cell"` on Category, Downloads, Popularity, Added columns
