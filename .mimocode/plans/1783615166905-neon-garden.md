## Changes

### 1. New Hook: `useAuditEcho`
**File:** `resources/js/hooks/useAuditEcho.ts`

Extract the WebSocket listener that subscribes to `audit.${audit_token}`, listens for `.AuditCompleted`, and shows toast notifications. Both pages have identical logic â€” the only difference is the callback for success (router.visit). Hook accepts an `onSuccess(report_url)` callback.

```ts
// Signature
useAuditEcho(auditToken: string, opts?: {
  onSuccess?: (reportUrl: string) => void;
  toastRef?: React.RefObject<Toast>;
}) => void
```

### 2. New Hook: `useAuditDialog`
**File:** `resources/js/hooks/useAuditDialog.ts`

Extract the audit dialog open/close/preselect state. Both pages manage `auditDialogVisible`, `auditMod`/`auditVersion`, and the open/close handlers with the same pattern.

```ts
// Signature
useAuditDialog() => {
  visible: boolean;
  preselectedMod: ModSearchResult | null;
  preselectedVersion: string | null;
  openForMod(mod: Mod): void;
  openForModVersion(mod: Mod, version: string): void;
  openNew(): void;
  close(): void;
}
```

### 3. New Component: `ModAvatar`
**File:** `resources/js/components/ui/ModAvatar.tsx`

Reusable avatar with image/letter fallback. Used in `NameColumn` (2.5rem, round) and `report.tsx` ModHeader (4rem, rounded square). Props: `image`, `name`, `title`, `size` (default "md"=2.5rem, "lg"=4rem), `shape` (default "circle", "square").

### 4. New Component: `ScoreBadge`
**File:** `resources/js/components/ui/ScoreBadge.tsx`

Displays a score number with computed color + optional ProgressBar. Used in `ScoreColumn` and report page stat cards / scanner scores.

```ts
// Signature
ScoreBadge({ score, size?: 'sm' | 'md' | 'lg', showBar?: boolean })
```
