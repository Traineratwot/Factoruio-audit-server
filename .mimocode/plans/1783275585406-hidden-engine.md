# Fix: Report button navigates to latest report version + outdated indicator

## Problem

The "Report" button in the mods table always navigates to `/report/mod/{name}` which defaults to `latest_version`. If the report exists only for an older version, the user sees "No audit report for version X" — an empty page.

## Goal

1. "Report" button → navigate to the latest version that has a report (not latest_version)
2. If report is outdated (not for latest version), show warning in both the table and the report page
3. Provide ability to request audit for the latest version from both places

---

## Changes

### 1. `app/Models/Mod.php` — Add helper method

```php
public function getLatestReportVersionAttribute(): ?string
{
    return $this->reports()
        ->orderByDesc('created_at')
        ->value('mod_version');
}
```

### 2. `app/Http/Controllers/ModController.php` — Default to latest report version

In `report()`, change the no-version fallback:

```php
// Before:
if (! $version) {
    $version = $mod->latest_version;
}

// After:
if (! $version) {
    $version = $mod->latest_report_version ?? $mod->latest_version;
}
```

Add `latest_version` to Inertia props:

```php
return Inertia::render('report', [
    ...
    'latest_version' => $mod->latest_version,
]);
```

### 3. `app/Http/Resources/ModResource.php` — Add outdated flag + fix report_url

```php
'latest_report_version' => $this->when(
    $this->relationLoaded('reports'),
    fn () => $this->latest_report_version,
),
'report_url' => $this->when(
    $this->relationLoaded('reports') && $this->reports->count() > 0,
    fn () => route('report.mod.version', [
        'mod' => $this->name,
        'version' => $this->latest_report_version,
    ]),
    fn () => route('report.mod', ['mod' => $this->name]),
),
```

### 4. `resources/js/types/mod.ts` — Add new fields

```ts
export interface Mod {
    ...
    latest_report_version?: string | null;
}
```

Add `latest_version` to report page props interface.

### 5. `resources/js/components/mods/columns/ActionColumn.tsx` — Outdated indicator

When `rowData.latest_report_version` exists and differs from `rowData.latest_version`, show a warning icon (exclamation triangle) next to the Report button with a tooltip explaining the report is for an older version.

### 6. `resources/js/pages/report.tsx` — Outdated warning banner

Add props: `latest_version: string | null`

When `report` exists and `current_version !== latest_version` and `latest_version` not in `reported_versions`, show a warning banner:

```
⚠ This report is for version X, but the latest version is Y.
[Audit latest version] button
```

The button opens AuditDialog pre-filled with `latest_version`.
