# Plan: Laravel Reverb + Audit Notifications

## Approach

Add Laravel Reverb WebSocket server and real-time toast notifications for audit start/completion.

**Channel strategy**: Public channels with a UUID token per audit request. Since audit requests come from unauthenticated users (rate-limited by IP), private channels with Laravel's auth won't work. Instead, `AuditController@store` generates a UUID `audit_token`, returns it in the JSON response, and the frontend subscribes to `audit.{token}`. Only the requester knows the token.

---

## Steps

### 1. Install packages

```bash
composer require laravel/reverb
bun add laravel-echo pusher-js
```

Run `php artisan reverb:install` to publish config files.

### 2. Environment variables

Add to `.env`:

```env
BROADCAST_CONNECTION=reverb

REVERB_APP_ID=audit-local
REVERB_APP_KEY=audit-local-key
REVERB_APP_SECRET=audit-local-secret
REVERB_HOST=localhost
REVERB_PORT=8080
REVERB_SCHEME=http

VITE_REVERB_APP_KEY="${REVERB_APP_KEY}"
VITE_REVERB_HOST="${REVERB_HOST}"
VITE_REVERB_PORT="${REVERB_PORT}"
VITE_REVERB_SCHEME="${REVERB_SCHEME}"
```

Prod overrides (in `.env.prod` or deploy config):
```env
REVERB_HOST=reverb.factorio-audit.example
REVERB_PORT=443
REVERB_SCHEME=https
```

### 3. Config files

- `config/broadcasting.php` тАФ will be created by `reverb:install`, verify `reverb` connection is default
- `config/reverb.php` тАФ will be created by `reverb:install`
- `bootstrap/app.php` тАФ add `channels: __DIR__.'/../routes/channels.php'` to `->withRouting()`

### 4. Backend events

Create `app/Events/AuditStarted.php`:
- Implements `ShouldBroadcast`
- Constructor: `string $auditToken, int $modId, string $modName, string $version`
- `broadcastOn()`: `new Channel("audit.{$this->auditToken}")`
- `broadcastAs()`: `'AuditStarted'`

Create `app/Events/AuditCompleted.php`:
- Implements `ShouldBroadcast`
- Constructor: `string $auditToken, int $modId, string $modName, string $version, ?string $reportUrl, ?string $error`
- `broadcastOn()`: `new Channel("audit.{$this->auditToken}")`
- `broadcastAs()`: `'AuditCompleted'`

### 5. Channel routes

Create `routes/channels.php` тАФ empty file (public channels need no authorization).

### 6. Modify AuditController (`app/Http/Controllers/AuditController.php`)

In `store()`:
- Generate `$auditToken = (string) Str::uuid()`
- Pass to `AuditJob::dispatch($mod->id, $version, $auditToken)`
- Return `'audit_token' => $auditToken` in JSON response

### 7. Modify AuditJob (`app/Jobs/AuditJob.php`)

- Add `public ?string $auditToken = null` constructor parameter
- In `handle()`:
  - Broadcast `AuditStarted` before `$mod->runAudit()`
  - Wrap `runAudit()` in try/catch
  - Broadcast `AuditCompleted` after (with `reportUrl` on success, `error` on failure)

### 8. Supervisor config (`docker/app/supervisord.conf`)

Uncomment lines 87-98 (remove `;` prefixes from the `[program:reverb]` block).

### 9. Docker Compose (`docker-compose.dev.yml`)

Add Reverb port to `app` service:
```yaml
ports:
  - "5273:5273"
  - "${PUB_APP_PORT}:9000"
  - "${PUB_REVERB_PORT:-8080}:8080"   # Reverb WebSocket
```

### 10. Frontend тАФ Echo setup

Create `resources/js/echo.ts`:
```typescript
import Echo from "laravel-echo";
import Pusher from "pusher-js";

window.Pusher = Pusher;

const echo = new Echo({
    broadcaster: "reverb",
    key: import.meta.env.VITE_REVERB_APP_KEY,
    wsHost: import.meta.env.VITE_REVERB_HOST,
    wsPort: Number(import.meta.env.VITE_REVERB_PORT) || 8080,
    wssPort: Number(import.meta.env.VITE_REVERB_PORT) || 443,
    forceTLS: (import.meta.env.VITE_REVERB_SCHEME ?? "https") === "https",
    enabledTransports: ["ws", "wss"],
});

export default echo;
```

Modify `resources/js/app.tsx` тАФ add `import "./echo";`

### 11. Frontend тАФ Toast notifications

Modify `resources/js/hooks/useAuditForm.ts`:
- Add `auditToken: string | null` to state
- In `submit()` success handler: store `data.audit_token`
- In `reset()`: clear `auditToken`
- Export `auditToken` from hook

Modify `resources/js/components/mods/AuditDialog.tsx`:
- Add `onAuditQueued?: (auditToken: string) => void` prop
- Call `onAuditQueued` after successful submit

Modify `resources/js/pages/welcome.tsx`:
- Add `<Toast ref={toast} position="bottom-right" />` from PrimeReact
- Add `auditToken` state, pass setter as `onAuditQueued` to AuditDialog
- Use `useEffect` to subscribe to `echo.channel(`audit.${auditToken}`)`
- Listen for `.AuditStarted` and `.AuditCompleted` events
- Show toast on each event
- On completed with `reportUrl`: show toast with clickable link
- Cleanup: `echo.leaveChannel()` on unmount or token change

Same pattern in `resources/js/pages/report.tsx` if audit can be triggered from there.

### 12. TypeScript types

Add to `resources/js/env.d.ts` or create `resources/js/types/echo.d.ts`:
```typescript
interface ImportMetaEnv {
    readonly VITE_REVERB_APP_KEY: string;
    readonly VITE_REVERB_HOST: string;
    readonly VITE_REVERB_PORT: string;
    readonly VITE_REVERB_SCHEME: string;
}
```
